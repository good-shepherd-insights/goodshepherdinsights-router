# Eddify Agent

A Mastra-powered HTTP server exposing **Eddify** — a project manager AI — via a clean REST API. Built with [`@mastra/core`](https://mastra.ai) and [`@openrouter/ai-sdk-provider`](https://openrouter.ai) for flexible LLM routing.

## Architecture

```
Client (Telegram / Frontend / CLI)
        │
        ▼
Node.js HTTP Server (@hono/node-server)
        │
        ▼
Mastra Hono Router (auto-generated endpoints)
        │
        ▼
Eddify Agent (@mastra/core Agent)
        │
        ├── System Instructions (SOUL.md + IDENTITY.md)
        │
        └── LLM: OpenRouter → openai/gpt-4o (or any model)
```

## Setup

### 1. Install dependencies

```bash
npm install
```

### 2. Configure environment

```bash
cp .env.example .env
```

Edit `.env` and set your OpenRouter API key:

```env
OPENROUTER_API_KEY=sk-or-v1-your-key-here
PORT=3000
```

Get your API key at [openrouter.ai/keys](https://openrouter.ai/keys).

### 3. Run the server

**Development (with hot reload):**
```bash
npm run dev
```

**Production:**
```bash
npm start
```

The server starts on `http://localhost:3000` by default.

## API Endpoints

All endpoints are auto-generated by Mastra's built-in Hono router.

### List agents
```bash
GET /api/agents
```

### Get Eddify agent details
```bash
GET /api/agents/eddify
```

### Generate a response (non-streaming)
```bash
curl -X POST http://localhost:3000/api/agents/eddify/generate \
  -H "Content-Type: application/json" \
  -d '{
    "messages": [
      { "role": "user", "content": "What is your role?" }
    ]
  }'
```

**Response:**
```json
{
  "text": "I am Eddify — a project manager AI...",
  "usage": { "promptTokens": 1200, "completionTokens": 150 }
}
```

### Stream a response (SSE)
```bash
curl -X POST http://localhost:3000/api/agents/eddify/stream \
  -H "Content-Type: application/json" \
  -d '{
    "messages": [
      { "role": "user", "content": "Help me plan a sprint." }
    ]
  }'
```

Returns a Server-Sent Events (SSE) stream.

### Memory threads (conversation history)

```bash
# Create a thread
curl -X POST http://localhost:3000/api/agents/eddify/memory/threads \
  -H "Content-Type: application/json" \
  -d '{ "resourceId": "user-123" }'

# Generate with thread context
curl -X POST http://localhost:3000/api/agents/eddify/generate \
  -H "Content-Type: application/json" \
  -d '{
    "messages": [{ "role": "user", "content": "What did we discuss?" }],
    "threadId": "<thread-id>",
    "resourceId": "user-123"
  }'
```

## Customisation

### Change the model

Edit [`src/mastra/agents/eddify.ts`](src/mastra/agents/eddify.ts) and change the model string:

```typescript
// Any model available on OpenRouter:
model: openrouter('anthropic/claude-3-5-sonnet')
model: openrouter('google/gemini-2.0-flash')
model: openrouter('meta-llama/llama-3.3-70b-instruct')
```

Browse all available models at [openrouter.ai/models](https://openrouter.ai/models).

### Update agent instructions

- **Personality & values**: Edit [`src/mastra/instructions/soul.ts`](src/mastra/instructions/soul.ts)
- **Role & capabilities**: Edit [`src/mastra/instructions/identity.ts`](src/mastra/instructions/identity.ts)
